¿Qué te llevas de esta sesión?

  • Plantillas de prompts para redactar y revisar textos académicos.
  • Un flujo de trabajo para revisión bibliográfica asistida por IA.
  • Buenas prácticas para evitar alucinaciones y errores de citación.
  • Checklist de uso ético y responsable en el contexto universitario.

Programa sugerido (120 min)

  1. Fundamentos útiles de IA generativa (15 min)
  2. Prompting aplicado al trabajo académico (30 min)
  3. Taller: revisión bibliográfica y síntesis (30 min)
  4. Automatizaciones sencillas (20 min)
  5. Riesgos, ética y evaluación crítica (15 min)
  6. Q&A + próximos pasos (10 min)

Recursos que puedes preparar ya

  • Un artículo reciente para practicar resumen crítico.
  • Un borrador (introducción o discusión) para reescritura guiada.
  • Una tabla de resultados para generar interpretaciones alternativas.
  • Una rúbrica de evaluación para convertirla en prompt de feedback.

Materiales de apoyo reutilizables (a partir de talleres previos)

1) Kit de prompts académicos

  • Prompt para resumen estructurado (objetivo, método, hallazgos, límites).
  • Prompt para crítica metodológica en economía aplicada.
  • Prompt para convertir notas dispersas en esquema de paper.

2) Flujo de revisión bibliográfica

  • Definición de pregunta + criterios de inclusión/exclusión.
  • Matriz de lectura (autor, método, datos, sesgos, aportación).
  • Síntesis final con trazabilidad y verificación manual.

3) Integridad y evaluación crítica

  • Checklist para detectar alucinaciones y referencias inventadas.
  • Regla de “doble comprobación” antes de entregar textos.
  • Registro de uso de IA para transparencia docente/investigadora.

Ejercicios prácticos en clase

  1. Reescribe una introducción en 3 niveles de profundidad.
  2. Compara dos prompts para discutir sesgos de respuesta.
  3. Extrae hipótesis testables de una revisión narrativa.
  4. Genera una rúbrica para evaluar trabajos con apoyo de IA.

Plantilla de prompt recomendada

Actúa como [rol].
Contexto: [tema y objetivo].
Material base: [texto/datos].
Tarea: [acción específica].
Formato de salida: [estructura].
Restricciones: [tono, longitud, citas, idioma].
Verificación: indica supuestos y posibles errores.

Despliegue recomendado

Para este proyecto (web estática + endpoint de chatbot con claves privadas), la opción más directa es Render.

  • Render: ideal para tu backend Node.js sin cambios de arquitectura.
  • Vercel/Netlify: muy buenas opciones si migras el endpoint a funciones serverless.

Importante: no pongas nunca las API keys en el frontend. Este proyecto ya está preparado para usarlas en servidor.
